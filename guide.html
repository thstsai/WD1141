期末專題繳交指南

按照以下步驟建立你的 GitHub Pages 個人作品集，並於截止日期前完成繳交。

🚀 六步驟完成繳交

1
建立 GitHub 帳號

前往 github.com 註冊帳號。建議使用學號或真實姓名作為帳號名稱，方便老師識別。完成後確認 email 驗證。

💡 已有 GitHub 帳號可直接跳到步驟 2。若要申請 GitHub Student Pack 可享有更多功能（非必須）。
2
建立 Portfolio Repository

登入後點擊右上角 「+」→ New repository。Repository name 填入 portfolio，設為 Public，勾選 Add a README file，然後按 Create repository。

⚠️ Repository 必須設為 Public，老師才能查閱你的作品與 commit 歷史。
3
啟用 GitHub Pages

進入 repository → 上方選單點 Settings → 左側選 Pages → Source 選 Deploy from a branch → Branch 選 main / root → Save。

約等待 1–2 分鐘後，你的網站網址將會是：

你的作品集網址 
https://{你的GitHub帳號}.github.io/portfolio
✅ 能看到網站（即使是空白頁面）就算完成第 1 週作業。請把網址記下來。
4
建立必要檔案結構

你的 repository 至少需要包含以下檔案：

portfolio/ ├── index.html # 作品集首頁（必要） ├── README.md # 專題說明（必要） ├── DEVLOG.md # 設計決策日誌（必要） ├── css/ │ └── style.css ├── js/ │ └── main.js └── images/ # 圖片資源（選配）

5
養成定期 Commit 習慣

Commit 記錄是 40% 歷程評量的核心依據。每次完成一個功能就 commit 一次，不要累積太多才一起提交。

✅ 好的 commit 訊息
add responsive navigation menu
fix: hero image broken on mobile
style: update color palette to teal
feat: add contact form with validation
refactor: split CSS into modules
❌ 不好的 commit 訊息
update files
first commit（完整網站）
done
fix bug
123
⚠️ 每週至少 3 次有意義 commit。若最後一週才大量 commit，或單次 commit 超過 300 行，將列為異常並於口試重點追問。

在電腦本機使用 Git 的基本指令：

Terminal / Git Bash 
# 第一次設定（只需做一次）
git config --global user.name "你的姓名"
git config --global user.email "your@email.com"

# Clone repository 到本機
git clone https://github.com/username/portfolio.git
cd portfolio

# 每次修改後的提交流程
git add .
git commit -m "feat: add hero section with animation"
git push

❓ 常見問題

GitHub Pages 網站更新後多久才會生效？ ▼
通常 push 之後 1–3 分鐘內生效。若超過 5 分鐘未更新，可至 repository → Actions 頁面查看部署狀態，或嘗試強制重新整理瀏覽器（Ctrl+Shift+R）。
可以使用 CSS Framework（Bootstrap、Tailwind）嗎？ ▼
可以使用，但口試時須能說明 Framework 的運作原理，以及為何選用該 Framework。建議第一個版本先用純 CSS 刻出基本樣式，再考慮加入 Framework，這樣 commit 歷史也更自然。
Repository 名稱一定要叫 "portfolio" 嗎？ ▼
不一定，名稱可自訂（如 my-portfolio、web-final 等）。只要確認 GitHub Pages 正確啟用、能公開存取即可。繳交表單中填入正確的網址和 repo 連結就沒問題。
若使用 GitHub 網頁直接編輯，commit 記錄算數嗎？ ▼
算數。每次在 GitHub 網頁上直接編輯並儲存，都會產生一筆 commit。但建議習慣使用本機 Git，因為開發效率更高，且口試可能問到本機開發流程。
AI 使用日誌要多詳細才夠？ ▼
至少記錄：使用哪個 AI 工具、輸入的 prompt 大意、AI 回應的核心內容、你做了哪些修改或調整、從中學到什麼。不需要貼完整對話，但要讓老師看出你有在思考 AI 的輸出，而非照單全收。
DEVLOG.md 與 AI 使用日誌是同一份文件嗎？ ▼
不同。DEVLOG.md 是你在 repository 裡維護的設計決策日誌，記錄每個功能的思考過程（包含是否用 AI）。AI 使用日誌是期末繳交的獨立報告，著重在整學期對 AI 工具使用的反思與總結。
