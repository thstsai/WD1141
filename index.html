🎯 專題主題：個人作品集網站

核心要求：以 GitHub Pages 部署真實作品

每位學生建立並部署「個人技術作品集」，網站本身就是作品。透過 GitHub 提交記錄追蹤學習歷程，難以由 AI 全程代勞。

GitHub Pages 部署 自訂網域（選配） 響應式設計 作品集頁面 履歷頁面 DEVLOG 設計日誌

📊 評量架構（總分 100 分）

40% 歷程評量（GitHub Commits）
  • Commit 頻率與訊息品質（是否有意義的描述）
  • 程式碼演進軌跡（從簡單到複雜，有修改痕跡）
  • 自訂 README.md 與 DEVLOG.md（說明設計決策）
防 AI 代勞邏輯：Commit 歷史若顯示「一次提交全部完成」，視為異常。要求每週至少 3 次有意義 commit，且須包含修改／修正記錄。
30% 現場口試（程式碼解說）
  • 隨機指定一段自己的程式碼，當場解釋設計邏輯
  • 要求修改一個小功能（如改變色彩主題、新增動態效果）
  • 回答「為什麼選擇這個做法，而非 X 做法？」
防 AI 代勞邏輯：即使用 AI 協助生成，也必須能解釋。問「這行 CSS 的 z-index 設多少？為什麼？」立刻辨別是否真懂。
20% 成果展示（Demo Day）
  • 5 分鐘展示網站功能與設計理念
  • 同學互評：可用性、創意、技術難度各 1 題
  • 即時回應同學提問
10% AI 使用反思報告

提交「AI 使用日誌」：記錄使用哪些工具、提了什麼 prompt、AI 輸出什麼、自己如何修改與理解。鼓勵使用 AI，但要求誠實記錄。

📅 執行時間軸（8 週）

第 1 週｜環境建置

建立 GitHub 帳號、初始化 repo、部署空白頁面到 GitHub Pages。繳交：能存取的網址。

第 2–3 週｜結構與樣式

HTML 架構完成、CSS 版型設計、響應式佈局。每週 checkpoint：3 次以上有意義 commit。

第 4–6 週｜功能開發

JavaScript 互動、表單、動態效果。期中檢核：老師 code review 一次，回饋須反映在後續 commit。

第 7 週｜完善與優化

效能、無障礙、SEO 基礎優化。提交 AI 使用日誌草稿。

第 8 週｜Demo Day + 口試

公開展示、同學互評、老師現場口試。繳交最終 AI 使用反思報告。

🤖 對 AI 工具的明確政策

✅ 鼓勵使用
  • 用 AI 查 bug 原因
  • 請 AI 解釋不懂的語法
  • 請 AI 建議改進方向
  • 用 AI 生成佔位文字 / 圖片
❌ 須自行完成
  • 網站整體架構設計
  • 版型與色彩決策
  • 功能邏輯規劃
  • AI 使用日誌的反思內容

核心理念：「能用 AI 完成」與「理解自己做了什麼」是兩件事。本課程評量的是後者。允許使用 AI 工具，但口試時須能解釋每一行自己提交的程式碼。
